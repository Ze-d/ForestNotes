树木排布算法 — 当前 vs 优化方案

P1：当前所有树均匀排列在菱形网格上，大 tag 不突出，相关 tag 可能分散

当前问题

当前 assignGridPositions — 菱形网格均匀排列

问题

  • 大 tag（重要 topic）和小 tag 随机混在一起，视觉权重不反映重要性
  • 高频共现的 tag 可能被排到岛的两端，藤蔓很长
  • 树大小差异大时可能穿插重叠

排布策略选择

决定树在岛上的位置规则（可多选组合）

1

按 noteCount 中心→边缘排列

大树在岛中心、小树在边缘。
实现：按 noteCount 排序，中心距离 = f(rank)，越重要越靠近中心

2

按共现聚类排列

藤蔓连接多的树聚集在一起。
实现：对 edge weight 做简单的力导向预布局，或按连通分量分组

3

螺旋/斐波那契排列

从中心向外螺旋分布，大树在螺旋起点。
特点：物理上均匀分布但视觉上中心汇聚，适合圆形岛

4

d3-force 预布局（推荐组合用）

在初始化时用 d3-force 跑 100 tick 做预布局。
用 collideradius = crownRadius + margin 保证不穿插。
然后固定位置，不再每帧跑物理

推荐组合

1 + 2 + 4 组合：
  1. 用 d3-force 预布局解决间距和穿插问题
  2. 初始位置按 noteCount 排序：大树靠近中心
  3. 用 link force 把高 weight 藤蔓的树拉近
  4. 跑 ~200 tick 收敛后固定，不再实时物理

这样既保留了 3D 场景的稳定感，又让视觉布局有意义
