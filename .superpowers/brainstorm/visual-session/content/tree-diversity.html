树冠形态多样性 — 当前 vs 目标

P0：当前所有树使用完全相同的锥形几何体，无法从外观区分不同 tag

问题：当前所有树一模一样

当前状态 — 所有树 = Cylinder + Cone × 3（仅大小不同）

Big tag (n=20)

Small tag (n=3)

Stale tag (n=3)

Dormant (n=3)

现状问题

  • 所有树都是「锥形三角形」轮廓
  • 只有大小和颜色不同，形态完全一致
  • 健康树和枯萎树看起来是"同一个树换了颜色"，不够自然

方案对比：如何让每棵树形态不同？

请选择树冠形态多样化的方向

A

几何体混合（推荐）

基于 hashSeed(id) 随机选择不同几何体组合：锥形、球形、椭球形、半球形。
优点：低多边形风格不变，代码改动小，每棵树立即不同
示例：松树→cone、阔叶树→sphere、灌木→hemisphere、混合→cone+sphere

B

参数化变异

同一锥形几何体，但通过随机参数（半径比、高度比、偏移、旋转）产生变化。
优点：代码量最小，不改几何体类型
缺点：变化幅度有限，远看可能还是差不多

C

完全随机生成（L-system）

每棵树用 Lindenmayer system 生成独一无二的枝干结构。
优点：最自然、每棵树都不同
缺点：实现复杂度高，与当前低多边形风格可能冲突
